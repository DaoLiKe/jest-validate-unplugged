# 模块简介 · jest-validate

> `jest-validate` 是 Jest 里负责「配置校验」的小模块，也是学习「输入校验 + 友好报错」的绝佳样本。

## 它解决什么问题
用户经常在配置里写错：键名拼错（`verboose` vs `verbose`）、类型不对（`maxWorkers: 50%` 应是字符串或数字）。`jest-validate` 在 Jest 启动时校验 `jest.config`，把这类错误用**人类可读**的方式报出来，而不是抛一堆堆栈。

## 核心 API（三件套）
- `validate(config, options)` — 主入口。传入用户配置 + 校验规则，返回 `{ isValid, hasDeprecationWarnings }`，出错时抛出 `ValidationError`。
- `ValidationError(name, message, options)` — 统一的错误类型，保证所有报错格式一致。
- `condition` / `format` / `utils` — 支撑函数：条件判断、消息排版、类型与字典工具。

## 为什么适合团队研读
1. **输入输出极清晰**：给一份配置，看它怎么校验、怎么报错。
2. **错误信息是产品**：`createDidYouMeanMessage` 用 Levenshtein 距离做「你是不是想写 xxx」，这是「把报错当用户体验来做」的范例。
3. **坏味道集中**：模块级可变状态、硬编码阈值、散落特例、恒为 true 的死字段——都在这里，方便团队练眼力。

## 一句话总结
> 一个小到能一次读完、却把「健壮校验 + 友好报错」讲透的模块——值得每个写配置 / 表单 / API 的人读一遍。
