# 项目简介 · jest-validate-unplugged

> 一份面向团队的、关于 Jest `jest-validate` 模块的「署名派生」学习仓库。

## 这是什么
`jest-validate-unplugged` 是从 [jestjs/jest](https://github.com/jestjs/jest) 中抽出的 **`jest-validate`** 配置校验模块，经过**零依赖重构**后用于团队内部研读。目标只有一个：用一份小而完整的真实代码，带团队看清「好代码长什么样、坏味道藏在哪里」。

## 为什么做它
- 团队要提升代码质量，最好的教材是**优秀开源项目的真实代码**，而不是抽象说教。
- `jest-validate` 体量小（单模块）、边界清晰（输入配置 → 输出校验结果），非常适合作为「代码评审训练样本」。
- 我们把 4 个典型**代码坏味道**标了出来，并给出了可落地的重构手法。

## 合规说明（重要）
这是 **jestjs/jest 的署名派生（attributed derivative）**，仅供学习：
- 原作者的 MIT License 与版权（Meta Platforms）已**原样保留** → `LICENSE`
- 署名与用途声明 → `ATTRIBUTION.md`
- 我们**没有**剥离原作者署名、也没有冒充原创

## 仓库里有什么
| 目录 / 文件 | 内容 |
| --- | --- |
| `original/` | Jest 原始源码逐字副本（validate / utils / errors） |
| `src/` | 零依赖重构版（内联 getType / leven / format，含中文注解） |
| `examples/` | 一个能直接跑的 demo（合法配置 / 拼写错误 / 类型错误三场景） |
| `STUDY_GUIDE.md` | 架构讲解 + 坏味道 + 重构策略 + 团队规范建议 |
| `模块简介.md` / `快速上手.md` | 本文档系列的其它简介 |
