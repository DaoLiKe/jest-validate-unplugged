# 快速上手 · 5 分钟读懂这份代码

> 不需要安装任何依赖，有 Node.js 直接跑。

## 1. 跑起来看看
```bash
node examples/usage.js
```
会演示三种场景：合法配置、拼写错误的键（`verboose`）、`maxWorkers` 类型错误。重点看**报错长什么样**。

## 2. 先读 original/，再读 src/
- `original/validate.ts` — Jest 原版，带 4 个外部依赖（`chalk` / `leven` / `pretty-format` / `@jest/get-type`）。
- `src/validate.js` — 我们的零依赖版，把上面 4 个依赖**内联**实现，逻辑一一对应，方便逐行对比。

## 3. 带着这 4 个问题去读
1. 模块里有没有「全局可变状态」？（提示：看 warnings 是怎么传的）
2. 有没有「写死的魔法数字 / 阈值」？（提示：`createDidYouMeanMessage` 的编辑距离上限）
3. 有没有「散落各处的特例判断」？
4. 有没有「永远为 true / 永远用不到」的死字段？

## 4. 练手（详见 STUDY_GUIDE.md）
- 给 `validate` 补几个单元测试
- 把 `isValid` 真正用起来（现在它算出来却没被消费）
- 基于这份代码，给团队写一条 ESLint 规则或 PR 互评清单
